# 팀빌더 시퀀스 다이어그램

## 1. 문서 개요

본 문서는 **팀빌더(Team Builder)** 기능에서 사용자가 포켓몬 5마리를 선택한 뒤, `팀 분석&추천` 버튼을 실행했을 때 화면, API, Graph DB, Vector DB, LLM, PostgreSQL이 어떤 순서로 상호작용하는지 정의한다.

화면에서는 버튼이 하나로 보이지만, 시스템 내부에서는 다음 두 단계가 분리되어 실행된다.

1. 팀 분석 파이프라인: 선택한 5마리 포켓몬의 약점, 방어 안정성, 기술 타입 커버리지를 분석한다.
2. 추천 파이프라인: 분석 결과를 바탕으로 6번째 포켓몬 후보를 추천하고 결과를 저장한다.

## 2. 참여 객체

| 객체 | 역할 |
|---|---|
| 사용자 | 팀빌더 화면에서 포켓몬 5마리를 선택하고 `팀 분석&추천`을 실행한다. |
| `teambuilding.py` | Streamlit 기반 팀빌더 화면이다. 선택 상태, 필터, 결과 렌더링을 담당한다. |
| `team_builder.py` | FastAPI 팀빌더 라우터이다. 분석/추천 요청을 서비스 계층으로 전달한다. |
| `team_rag_service.py` | LangGraph 기반 Hybrid RAG 실행 진입점이다. |
| `workflow.py` | Graph 조회, Vector 검색, Hybrid Score 계산, LLM 해설 생성을 순차 실행한다. |
| Neo4j Graph DB | 포켓몬 타입, 약점, 저항, 기술 커버리지, 추천 후보 계산에 필요한 관계 데이터를 제공한다. |
| Vector DB | 포켓몬 설명, 기술 설명, 특성 설명 등 텍스트 근거를 검색한다. |
| LLM | Graph 근거와 Vector 근거를 바탕으로 자연어 분석/추천 해설을 생성한다. |
| PostgreSQL | 최종 팀 분석 및 추천 결과를 `team_build_logs` 테이블에 저장한다. |

## 3. 전체 처리 흐름

```mermaid
sequenceDiagram
    autonumber
    actor User as 사용자
    participant FE as teambuilding.py
    participant API as team_builder.py
    participant RAG as team_rag_service.py
    participant WF as workflow.py
    participant GDB as Neo4j Graph DB
    participant VDB as Vector DB
    participant LLM as LLM
    participant PDB as PostgreSQL

    User->>FE: 팀빌더 화면 진입
    FE->>API: GET /api/v1/team-builder/pokemon-options
    API->>GDB: 포켓몬 목록/타입/특성 조회
    GDB-->>API: 포켓몬 카드 데이터 반환
    API-->>FE: 포켓몬 목록 JSON 반환
    FE-->>User: 검색/필터 가능한 포켓몬 카드 표시

    User->>FE: 포켓몬 5마리 선택
    FE-->>User: 선택한 포켓몬 영역 갱신

    User->>FE: 팀 분석&추천 실행
    FE->>FE: 선택 수 5마리 검증

    rect rgb(235, 248, 255)
        Note over FE,LLM: 1단계: 팀 분석 파이프라인
        FE->>API: POST /api/v1/team-builder/rag-analyze
        API->>RAG: request_type=analysis
        RAG->>WF: Hybrid RAG workflow 실행
        WF->>GDB: 팀 약점/강점/기술 커버리지 조회
        WF->>VDB: 분석 근거 문서 검색
        WF->>LLM: Graph + Vector 근거로 분석 해설 생성
        LLM-->>WF: 분석 final_answer 반환
        WF-->>RAG: 분석 결과 state 반환
        RAG-->>API: 분석 결과 반환
        API-->>FE: 분석 JSON 반환
    end

    rect rgb(240, 253, 244)
        Note over FE,PDB: 2단계: 추천 및 저장 파이프라인
        FE->>API: POST /api/v1/team-builder/rag-recommend
        Note right of FE: pokemon_ids + analysis_result 전달
        API->>RAG: request_type=recommendation
        RAG->>WF: Hybrid RAG workflow 실행
        WF->>GDB: 6번째 포켓몬 후보 계산
        WF->>VDB: 추천 근거 문서 검색
        WF->>WF: graph_score + vector_score 결합
        WF->>LLM: Graph + Vector 근거로 추천 해설 생성
        LLM-->>WF: 추천 final_answer 반환
        WF-->>RAG: 추천 결과 state 반환
        RAG-->>API: 추천 결과 반환
        API->>PDB: team_build_logs 저장
        PDB-->>API: team_build_log_id 반환
        API-->>FE: 추천 결과 + 저장 ID 반환
    end

    FE-->>User: 분석 결과와 추천 결과를 같은 화면에 표시
```

## 4. 포켓몬 목록 조회 및 선택 흐름

```mermaid
sequenceDiagram
    autonumber
    actor User as 사용자
    participant FE as teambuilding.py
    participant API as team_builder.py
    participant GDB as Neo4j Graph DB

    User->>FE: 팀빌더 페이지 접속
    FE->>API: GET /api/v1/team-builder/pokemon-options
    API->>GDB: Pokemon, Type, Ability 조회
    GDB-->>API: 이름, 이미지, 타입, 특성 반환
    API-->>FE: 포켓몬 목록 반환
    FE-->>User: 카드 목록 표시

    loop 검색/필터 변경
        User->>FE: 검색어, 도감번호, 타입, 특성, 지방 조건 입력
        FE->>FE: 화면 내부 필터링
        FE-->>User: 필터링된 카드 목록 표시
    end

    User->>FE: 포켓몬 카드 선택
    FE->>FE: 선택 목록에 포켓몬 ID 추가
    FE-->>User: 선택한 포켓몬 영역 갱신

    alt 선택 수 5마리 미만
        FE-->>User: 팀 분석&추천 버튼 비활성화
    else 선택 수 5마리
        FE-->>User: 팀 분석&추천 버튼 활성화
    end
```

## 5. 팀 분석&추천 실행 시퀀스

```mermaid
sequenceDiagram
    autonumber
    actor User as 사용자
    participant FE as teambuilding.py
    participant API as team_builder.py
    participant RAG as team_rag_service.py
    participant WF as LangGraph Workflow
    participant GDB as Neo4j Graph DB
    participant VDB as Vector DB
    participant LLM as LLM
    participant PDB as PostgreSQL

    User->>FE: 팀 분석&추천 실행
    FE->>FE: 선택 포켓몬 5마리 검증

    FE->>API: POST /api/v1/team-builder/rag-analyze
    Note right of FE: payload = pokemon_ids, user_id
    API->>RAG: run_team_rag(request_type="analysis")
    RAG->>WF: initial_state 생성 후 invoke

    WF->>WF: supervisor - 요청 검증
    WF->>WF: select_graph_tool - 분석 도구 선택
    WF->>GDB: execution_graph_tool - 팀 분석 쿼리 실행
    GDB-->>WF: weak_types, resistant_types, move_type_coverage 반환

    WF->>VDB: vector_search - 분석 근거 검색
    VDB-->>WF: evidence_documents 반환

    WF->>WF: hybrid_scorer - 분석 점수 결합
    Note right of WF: 현재 코드 기준 분석 가중치 = Graph 60%, Vector 40%

    WF->>LLM: answer_generator - 분석 해설 프롬프트 전달
    Note right of WF: AI 해설 근거 비율 = Graph 50%, Vector 50%
    LLM-->>WF: 분석 final_answer 반환
    WF-->>RAG: 분석 결과 state 반환
    RAG-->>API: 분석 결과 반환
    API-->>FE: 분석 JSON 반환

    FE->>API: POST /api/v1/team-builder/rag-recommend
    Note right of FE: pokemon_ids, limit, user_id, analysis_result 전달
    API->>RAG: run_team_rag(request_type="recommendation")
    RAG->>WF: initial_state 생성 후 invoke

    WF->>WF: supervisor - 요청 검증
    WF->>WF: select_graph_tool - 추천 도구 선택
    WF->>GDB: execution_graph_tool - 후보 추천 쿼리 실행
    GDB-->>WF: 후보 포켓몬, graph_score, 보완 근거 반환

    WF->>VDB: vector_search - 추천 근거 검색
    VDB-->>WF: 후보별 텍스트 근거 반환

    WF->>WF: hybrid_scorer - hybrid_score 계산
    Note right of WF: 추천 가중치 = Graph 70%, Vector 30%

    WF->>LLM: answer_generator - 추천 해설 프롬프트 전달
    Note right of WF: AI 해설 근거 비율 = Graph 50%, Vector 50%
    LLM-->>WF: 추천 final_answer 반환
    WF-->>RAG: 추천 결과 state 반환
    RAG-->>API: 추천 결과 반환

    API->>API: 추천 포켓몬 ID와 결론 문장 추출
    API->>PDB: team_build_logs INSERT
    PDB-->>API: team_build_log_id 반환

    API-->>FE: 추천 결과 + 저장 ID 반환
    FE-->>User: 분석 카드, 추천 카드, AI 종합 해설 표시
```

## 6. 예외 처리 시퀀스

```mermaid
sequenceDiagram
    autonumber
    actor User as 사용자
    participant FE as teambuilding.py
    participant API as team_builder.py
    participant GDB as Neo4j Graph DB
    participant LLM as LLM

    alt 선택 포켓몬이 5마리가 아님

    else Graph DB 연결 실패
        API->>GDB: Neo4j 쿼리 실행
        GDB--x API: 연결 실패
        API-->>FE: 오류 응답
        FE-->>User: 분석/추천 실패 안내
    else LLM 응답 실패
        API->>LLM: AI 해설 생성 요청
        LLM--x API: timeout 또는 인증 실패
        API-->>FE: 오류 응답
        FE-->>User: AI 해설 생성 실패 안내
    end
```

## 7. 가중치 적용 위치

| 단계 | 적용 위치 | Graph DB | Vector DB | 설명 |
|---|---|---:|---:|---|
| 팀 분석 | `hybrid_scorer.py` | 60% | 40% | 팀 약점/강점 분석 결과와 텍스트 근거를 결합한다. |
| 포켓몬 추천 | `hybrid_scorer.py` | 70% | 30% | 추천 후보 순위를 산정한다. |
| AI 해설 | `answer_generator.py` | 50% | 50% | 최종 설명 문장 생성 시 두 근거를 균형 있게 사용한다. |

## 8. 구현 파일 매핑

| 흐름 | 주요 파일 |

| 결과 저장 | `backend/crud.py`, `backend/models.py`, `backend/schemas.py` |

## 9. 검토 포인트

- 화면 버튼명은 `팀 분석&추천` 하나로 표현한다.
- 내부 API는 분석과 추천을 분리하여 호출할 수 있다.
- 분석 결과는 추천 요청의 입력 근거로 재사용한다.
- 최종 저장은 추천 결과까지 생성된 뒤 `team_build_logs`에 한 번 수행한다.
- 저장 결과에는 선택 포켓몬, 분석 결과, 분석 결론, 추천 포켓몬, 추천 결과, 추천 결론이 포함되어야 한다.
