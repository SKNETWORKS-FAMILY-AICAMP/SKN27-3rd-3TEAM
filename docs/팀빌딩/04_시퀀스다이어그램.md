# 팀빌더 시퀀스 다이어그램

## 1. 문서 개요

본 문서는 **팀빌더 기능**에서 사용자의 화면 조작이 프론트엔드, 백엔드 API, Graph DB, Vector DB, LLM, PostgreSQL 저장까지 어떤 순서로 이어지는지 정의한다.

팀빌더의 핵심 흐름은 다음과 같다.

1. 사용자가 팀빌더 화면에 진입한다.
2. 포켓몬 목록을 조회하고 검색/필터를 적용한다.
3. 포켓몬 5마리를 선택한다.
4. 덱 분석 버튼을 눌러 Hybrid RAG 기반 분석 결과를 받는다.
5. 추천 받기 버튼을 눌러 6번째 포켓몬 후보를 추천받는다.
6. 추천 결과가 생성되면 분석/추천 결과를 PostgreSQL에 저장한다.

## 2. 참여 객체

| 객체 | 역할 |
|---|---|
| 사용자 | 팀빌더 화면에서 포켓몬을 선택하고 분석/추천을 요청한다. |
| `teambuilding.py` | Streamlit 기반 팀빌더 화면이다. |
| `team_builder.py` | FastAPI 팀빌더 라우터이다. |
| `team_rag_service.py` | LangGraph 기반 Hybrid RAG 실행 진입점이다. |
| `workflow.py` | supervisor, graph tool, vector search, hybrid scorer, answer generator를 순차 실행한다. |
| Neo4j Graph DB | 포켓몬, 타입, 기술, 상성 관계를 조회한다. |
| Vector DB | 포켓몬 설명/근거 문서를 검색한다. |
| LLM | Graph + Vector 근거를 바탕으로 AI 종합 해설을 생성한다. |
| PostgreSQL | 팀빌더 분석/추천 결과를 `team_build_logs`에 저장한다. |

## 3. 전체 처리 흐름

```mermaid
sequenceDiagram
    autonumber
    participant User as 사용자
    participant FE as teambuilding.py
    participant API as team_builder.py
    participant RAG as team_rag_service.py
    participant WF as workflow.py
    participant GDB as Neo4j Graph DB
    participant VDB as Vector DB
    participant LLM as LLM
    participant PDB as PostgreSQL

    User->>FE: 팀빌더 화면 진입
    FE->>API: GET /api/v1/team-builder/pokemon-options
    API->>GDB: 포켓몬 목록/타입/특성 조회
    GDB-->>API: 포켓몬 카드 데이터 반환
    API-->>FE: 포켓몬 목록 JSON 반환
    FE-->>User: 검색/필터 가능한 포켓몬 카드 표시

    User->>FE: 포켓몬 5마리 선택
    User->>FE: 덱 분석 클릭
    FE->>API: POST /api/v1/team-builder/rag-analyze
    API->>RAG: request_type=analysis
    RAG->>WF: Hybrid RAG workflow 실행
    WF->>GDB: 팀 약점/강점/기술 커버리지 조회
    WF->>VDB: 분석 근거 문서 검색
    WF->>LLM: Graph + Vector 근거로 분석 해설 생성
    LLM-->>WF: final_answer 반환
    WF-->>RAG: 분석 결과 반환
    RAG-->>API: 분석 결과 반환
    API-->>FE: 덱 분석 JSON 반환
    FE-->>User: 분석 카드와 AI 종합 해설 표시

    User->>FE: 추천 받기 클릭
    FE->>API: POST /api/v1/team-builder/rag-recommend
    API->>RAG: request_type=recommendation
    RAG->>WF: Hybrid RAG workflow 실행
    WF->>GDB: 6번째 포켓몬 후보 계산
    WF->>VDB: 추천 근거 문서 검색
    WF->>LLM: Graph + Vector 근거로 추천 해설 생성
    LLM-->>WF: final_answer 반환
    WF-->>RAG: 추천 결과 반환
    RAG-->>API: 추천 결과 반환
    API->>PDB: team_build_logs 저장
    PDB-->>API: 저장 ID 반환
    API-->>FE: 추천 결과 + team_build_log_id 반환
    FE-->>User: 추천 1~3순위와 AI 추천 해설 표시
```

## 4. 포켓몬 목록 조회 및 선택 흐름

```mermaid
sequenceDiagram
    autonumber
    participant User as 사용자
    participant FE as teambuilding.py
    participant API as team_builder.py
    participant GDB as Neo4j Graph DB

    User->>FE: 팀빌더 페이지 접속
    FE->>API: GET /api/v1/team-builder/pokemon-options
    API->>GDB: Pokemon, Type, Ability 조회
    GDB-->>API: 포켓몬 이름, 이미지, 타입, 특성 반환
    API-->>FE: 포켓몬 목록 반환
    FE-->>User: 카드 목록 표시

    loop 검색/필터 변경
        User->>FE: 검색어, 도감번호, 타입, 특성, 지방 조건 입력
        FE->>FE: 화면 내부 필터링
        FE-->>User: 필터링된 카드 목록 표시
    end

    User->>FE: 포켓몬 선택
    FE->>FE: 선택 목록에 추가
    FE-->>User: 선택 대기 영역 업데이트

    alt 선택 수가 5마리 미만
        FE-->>User: 덱 분석/추천 버튼 비활성화
    else 선택 수가 5마리
        FE-->>User: 덱 분석 버튼 활성화
    end
```

## 5. 덱 분석 시퀀스

```mermaid
sequenceDiagram
    autonumber
    participant User as 사용자
    participant FE as teambuilding.py
    participant API as team_builder.py
    participant RAG as team_rag_service.py
    participant WF as LangGraph Workflow
    participant GDB as Neo4j Graph DB
    participant VDB as Vector DB
    participant LLM as LLM

    User->>FE: 덱 분석 클릭
    FE->>FE: 선택 포켓몬 5마리 검증
    FE->>API: POST /api/v1/team-builder/rag-analyze
    Note right of FE: payload = pokemon_ids, user_id

    API->>RAG: run_team_rag(request_type="analysis")
    RAG->>WF: initial_state 생성 후 invoke

    WF->>WF: supervisor - 요청 검증
    WF->>WF: select_graph_tool - 분석 도구 선택
    WF->>GDB: execution_graph_tool - 팀 분석 쿼리 실행
    GDB-->>WF: weak_types, resistant_types, move_type_coverage 반환

    WF->>VDB: vector_search - 분석 근거 검색
    VDB-->>WF: evidence_documents 반환

    WF->>WF: evaluate_with_llm - Graph/Vector 근거 결합
    WF->>WF: hybrid_scorer - 분석 정책 적용
    Note right of WF: 현재 코드 기준 분석 가중치 = Graph 60%, Vector 40%

    WF->>LLM: answer_generator - 분석 해설 프롬프트 전달
    LLM-->>WF: final_answer 반환

    WF-->>RAG: 최종 분석 state 반환
    RAG-->>API: 분석 결과 반환
    API-->>FE: 분석 JSON 반환
    FE-->>User: 덱 총평, 핵심 위험, 팀 강점, AI 종합 해설 표시
```

## 6. 포켓몬 추천 및 저장 시퀀스

```mermaid
sequenceDiagram
    autonumber
    participant User as 사용자
    participant FE as teambuilding.py
    participant API as team_builder.py
    participant RAG as team_rag_service.py
    participant WF as LangGraph Workflow
    participant GDB as Neo4j Graph DB
    participant VDB as Vector DB
    participant LLM as LLM
    participant PDB as PostgreSQL

    User->>FE: 추천 받기 클릭

    alt 덱 분석 결과가 없음
        FE-->>User: 덱 분석을 먼저 실행하라는 안내 표시
    else 덱 분석 결과가 있음
        FE->>API: POST /api/v1/team-builder/rag-recommend
        Note right of FE: payload = pokemon_ids, limit, user_id, analysis_result

        API->>RAG: run_team_rag(request_type="recommendation")
        RAG->>WF: initial_state 생성 후 invoke

        WF->>WF: supervisor - 요청 검증
        WF->>WF: select_graph_tool - 추천 도구 선택
        WF->>GDB: execution_graph_tool - 후보 추천 쿼리 실행
        GDB-->>WF: 후보 포켓몬, graph_score, 보완 약점 반환

        WF->>VDB: vector_search - 추천 근거 검색
        VDB-->>WF: 후보별 설명 근거 반환

        WF->>WF: hybrid_scorer - hybrid_score 계산
        Note right of WF: 추천 가중치 = Graph 70%, Vector 30%

        WF->>LLM: answer_generator - 추천 해설 프롬프트 전달
        Note right of WF: AI 해설 근거 비율 = Graph 50%, Vector 50%
        LLM-->>WF: final_answer 반환

        WF-->>RAG: 추천 결과 state 반환
        RAG-->>API: 추천 결과 반환

        API->>API: 추천 포켓몬 ID와 결론 문장 추출
        API->>PDB: team_build_logs INSERT
        PDB-->>API: team_build_log_id 반환

        API-->>FE: 추천 결과 + 저장 ID 반환
        FE-->>User: 추천 1~3순위, 추천 점수, AI 추천 해설 표시
    end
```

## 7. 예외 처리 시퀀스

```mermaid

    else LLM 응답 실패

        FE-->>User: AI 해설 생성 실패 안내
```

## 8. 가중치 적용 위치

| 단계 | 적용 위치 | Graph DB | Vector DB | 설명 |
|---|---|---:|---:|---|
| 덱 분석 | `hybrid_scorer.py` | 60% | 40% | 분석 결과와 근거 문서 결합 |
| 포켓몬 추천 | `hybrid_scorer.py` | 70% | 30% | 추천 후보 순위 산정 |
| AI 해설 | `answer_generator.py` | 50% | 50% | 설명 문장 생성 시 근거 균형 |

## 10. 검토 포인트

- 사용자는 반드시 포켓몬 5마리를 선택한 뒤 덱 분석을 실행해야 한다.
- 추천은 덱 분석 결과가 존재하는 상태에서만 실행되어야 한다.
- 분석 API는 화면 표시용 결과만 반환하고, 최종 저장은 추천 완료 시점에 수행한다.
- 추천 결과에는 1~3순위 포켓몬, 추천 점수, 추천 이유, AI 종합 해설이 포함되어야 한다.
- 저장 결과에는 선택 포켓몬, 분석 결과, 분석 결론, 추천 포켓몬, 추천 결과, 추천 결론이 포함되어야 한다.
