# 🤖 LangGraph 기반 지능형 배틀 봇 개발 계획

이 문서는 기존의 무작위 의사결정 기반 `BattleBot`을 LLM과 LangGraph를 활용한 지능형 전략 봇으로 고도화하기 위한 상세 계획을 담고 있습니다.

## 1. 개요 및 평가

### 1.1 현재 상황
- 봇은 현재 15%의 확률로 교체하거나, 4개의 기술 중 하나를 무작위로 선택하는 단순한 로직을 가지고 있습니다.
- 배틀의 상성, 현재 HP 상태, 남은 포켓몬 등을 전혀 고려하지 못하고 있습니다.

### 1.2 제안 구상 (LangGraph 도입)
- **순차적 추론**: 배틀의 상태를 입력받아 '상황 분석' -> '상대 행동 예측' -> '최적 행동 결정'의 과정을 거칩니다.
- **상태 관리**: 랭그래프의 `State`를 통해 현재 턴의 정보를 유지하며 다단계 추론을 수행합니다.
- **평가**: 매우 적절한 구상입니다. 포켓몬 배틀은 턴제이며 정보의 가중치가 턴마다 변하므로, 그래프 구조를 통한 추론 과정이 봇의 지능을 비약적으로 높일 수 있습니다.

## 2. 시스템 아키텍처

### 2.1 프로세스 흐름
1.  **배틀 페이지**: 턴이 시작되면 `BattleBot` 객체를 호출합니다.
2.  **BattleBot**: 전략이 'llm'일 경우 LangGraph 워크플로우를 실행합니다.
3.  **LangGraph**: 정의된 노드들을 따라 추론을 수행하고 최종 행동을 결정합니다.
4.  **반환**: 결정된 행동(`move` 또는 `switch`)을 다시 배틀 시스템에 전달합니다.

### 2.2 LangGraph 노드 구성
- **Node 1: Context Builder**: `BattlePokemon` 객체 및 배틀 데이터를 LLM 친화적인 텍스트(JSON/Markdown)로 요약합니다.
- **Node 2: Strategic Analyzer**: 타입 상성, 위협 요소, 교체 필요성을 분석하는 추론 노드입니다.
- **Node 3: Action Selector**: 분석된 내용을 바탕으로 유효한 행동 리스트 중 최적의 하나를 선택합니다.

## 3. 개발 로드맵

### 1단계: 기반 구축 (LLM 인프라)
- `frontend/battle/llm.py` 파일 내에 LangChain/LangGraph 설정.
- 배틀 상황 요약을 위한 프롬프트 템플릿 설계.

### 2단계: 워크플로우 구현 (LangGraph)
- 배틀 전용 `State` 정의 및 그래프 노드 연결.
- LLM의 답변을 배틀 시스템이 처리할 수 있는 `dict` 형식으로 파싱하는 파서(Parser) 구현.

### 3단계: 시스템 통합
- `trainer_bot.py`와 LangGraph 워크플로우 연결.
- 실제 배틀 테스트를 통한 전략 적합성 검증.

### 4단계: 고도화 (RAG 및 페르소나)
- 관장마다 서로 다른 전략(페르소나) 부여.
- 외부 지식 베이스(RAG)를 통한 상성 및 도감 정보 활용 고도화.
