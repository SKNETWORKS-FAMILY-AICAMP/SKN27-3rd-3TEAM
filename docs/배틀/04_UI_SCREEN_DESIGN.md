# 화면 설계서 (UI/UX Design)

## 개요

배틀 시스템의 프론트엔드는 **Streamlit** 을 사용하여 구현되었습니다. 다음은 배틀 화면의 구성과 사용자 인터렉션을 설명합니다.

## 8. 반응형 디자인 고려사항

### 8.1 화면 크기별 최적화

| 구분 | 해상도 | 배치 | 특징 |
|------|--------|------|------|
| 데스크톱 | 1920×1080+ | 2열 + 상세정보 | 풀 정보 표시 |
| 태블릿 | 1024×768 | 1열 + 접기 | 터치 최적화 |
| 모바일 | 480×800 | 스택 | 세로 스크롤 |

### 8.2 성능 최적화

```
로딩 시간 목표:
- 배틀 시작: < 2초
- 턴 처리: < 1초
- 포켓몬 선택: < 500ms

캐싱 전략:
- 포켓몬 데이터: 세션 유지
- 이미지: 브라우저 캐시 (1시간)
- 타입 상성: 메모리 (한 번만 로드)
```

## 9. 접근성 (Accessibility)

### 9.1 색상 대비

- 텍스트: WCAG AA 준수 (contrast ratio ≥ 4.5:1)
- HP 바: 색맹 사용자를 위해 패턴 추가
- 상태이상: 아이콘 + 텍스트 병행

### 9.2 키보드 네비게이션

```
Tab     : 다음 요소 선택
Shift+Tab: 이전 요소 선택
Enter   : 선택/제출
Space   : 체크박스 토글
Esc     : 취소/닫기
```

### 9.3 음성 지원

- 중요 메시지 읽기 제공
- 배틀 로그 스크린 리더 호환

## 10. 애니메이션 및 피드백

### 10.1 전환 효과

```
기술 사용 애니메이션:
1. 포켓몬 지정 (0.2초)
2. 기술 이름 표시 (0.3초)
3. 효과 애니메이션 (0.8초)
4. 데미지 표시 (0.5초)
5. HP 바 감소 (0.6초)

총 소요시간: ~2.4초 / 턴
```

### 10.2 피드백

| 액션 | 피드백 |
|------|--------|
| 기술 선택 | 버튼 강조 + 효과음 |
