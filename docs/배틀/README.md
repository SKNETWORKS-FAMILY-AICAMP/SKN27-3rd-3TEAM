# 배틀 시스템 문서 가이드

## 📑 문서 목록

### 1️⃣ **데이터베이스 설계** (`01_DATABASE_DESIGN.md`)
**대상:** 백엔드 개발자, 데이터 엔지니어  
**내용:**
- RDB (PostgreSQL) 스키마 설계
- 그래프 DB (Neo4j) 구조 (현재 미활용)
- 배틀에 필요한 데이터 흐름
- 데이터베이스 선택 결정사항

---

### 2️⃣ **데이터 수집 및 전처리** (`02_DATA_COLLECTION_PREPROCESSING.md`)
**대상:** 데이터 엔지니어, ML 엔지니어  
**내용:**
- PokéAPI에서 데이터 수집 프로세스
- 누락 데이터 보충 (수작업 매핑)
- 한글화 과정
- 데이터 검증 및 품질 확인

---

### 3️⃣ **배틀 시퀀스 다이어그램** (`03_BATTLE_SEQUENCE_DIAGRAM.md`)
**대상:** 전체 팀 (시스템 이해 목적)  
**내용:**
- 전체 배틀 흐름 (High-Level)
- 포켓몬 로드 프로세스
- 1턴 처리 상세 단계
- 기술 처리 로직
- 데미지 계산 공식

---

### 4️⃣ **화면 설계서** (`04_UI_SCREEN_DESIGN.md`)
**대상:** 프론트엔드 개발자, 디자이너, PM  
**내용:**
- 배틀 UI 구성
- 기술 선택 인터페이스
- 상태이상 표시 방식
- 배틀 종료 화면
- 팀 빌더 UI

---

### 5️⃣ **테스트 시나리오 및 결과** (`05_TEST_SCENARIOS_RESULTS.md`)
**대상:** QA, 개발자, PM  
**내용:**
- 배틀 로직 테스트 (5개 시나리오)
- 데이터 무결성 테스트 (3개)
- API 엔드포인트 테스트 (3개)
- UI/UX 테스트 (3개)
