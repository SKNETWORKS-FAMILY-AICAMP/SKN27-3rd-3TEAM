# 요구사항 명세서 (Software Requirements Specification)

**프로젝트명:** 포켓몬 AI 챗봇 (Pokémon AI Chatbot)  
**문서 버전:** v1.2  
**작성일:** 2025-05-14  
**최종 수정:** 2025-05-14 (Hybrid Search 도입, CRAG 추가, web_search 제거, MAX_TOOL_CALLS 5로 상향)  
**상태:** 초안 (Draft)

---

## 1. 프로젝트 개요

### 1.1 목적

PostgreSQL(정형 데이터), PGVector(벡터 검색), Neo4j(그래프 DB)를 통합한 LangGraph 기반 Tool-calling Agent를 구축하여, 사용자가 자연어로 포켓몬 정보를 탐색할 수 있는 지능형 챗봇 시스템을 개발한다.

### 1.2 범위

| 구분 | 포함 | 제외 |
|------|------|------|
| 백엔드 | LangGraph Agent, Tool 함수, DB 연동 | 프론트엔드 UI 구현 |
| 데이터 | 포켓몬 1~9세대 정형·비정형 데이터 | 실시간 게임 데이터 |
| 검색 | SQL / 벡터+BM25+pg_trgm 하이브리드 / 그래프 | 이미지 검색, 웹 검색 |
| 대화 | 멀티턴 세션 저장·복원 | 음성 인터페이스 |

### 1.3 용어 정의

| 용어 | 정의 |
|------|------|
| Agent | LLM이 툴을 자율 선택·실행하는 LangGraph 기반 실행 단위 |
| Tool | Agent가 호출할 수 있는 개별 기능 함수 |
| MMR | Maximal Marginal Relevance — 유사도와 다양성을 동시에 고려한 벡터 검색 알고리즘 |
| BM25 | 단어 빈도 기반 키워드 검색 알고리즘 (rank_bm25, 프로세스 메모리 상주) |
| pg_trgm | PostgreSQL 트라이그램 유사도 검색 확장 |
| RRF | Reciprocal Rank Fusion — 여러 검색 채널 결과를 순위 역수 합산으로 통합하는 알고리즘 |
| Hybrid Search | Vector(MMR) + BM25 + pg_trgm 세 채널을 RRF로 결합한 복합 검색 방식 |
| CRAG | Corrective RAG — 툴 실행 후 LLM이 결과 관련성을 평가하고, 관련 없으면 재검색을 유도하는 패턴 |
| Query Rewriting | 사용자 질문의 핵심 의도를 유지하며 검색 키워드를 명확히 재작성하는 전처리 단계 |
| Flavor Text | 포켓몬 도감에 수록된 버전별 설명 텍스트 |
| Embedding | 텍스트를 고차원 벡터로 변환한 수치 표현 |
| Hallucination | LLM이 사실과 다른 정보를 생성하는 현상 |
| AGAINST | Neo4j 관계 타입. `(Pokemon)-[:AGAINST {multiplier}]->(Type)` 형태로 포켓몬이 특정 공격 타입을 받을 때의 실제 배율(듀얼 타입 복합 계산 포함)을 저장 |
| 듀얼 타입 | 포켓몬이 두 가지 타입을 동시에 보유하는 경우. 방어 배율이 두 타입의 곱으로 결정됨 (예: 4배, 0.25배) |
| KNOWN_EVO_CONDITIONS | Neo4j에 미수록된 진화 조건(친밀도 진화 등)을 보완하는 하드코딩 fallback 맵 |

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 자연어 질의응답

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-01 | 사용자는 자연어로 포켓몬 이름, 타입, 스탯, 세대, 포획률 등 정형 정보를 질의할 수 있어야 한다. | 필수 |
| FR-02 | 사용자는 "귀여운", "강한 느낌" 등 추상적 묘사로 포켓몬을 검색할 수 있어야 한다. | 필수 |
| FR-03 | 사용자는 특정 포켓몬의 진화 경로와 진화 조건(레벨/아이템/친밀도)을 질의할 수 있어야 한다. | 필수 |
| FR-04 | 사용자는 타입 자체의 공격·방어 상성(강점/약점/무효/저항/면역 포켓몬 예시)을 질의할 수 있어야 한다. | 필수 |
| FR-04-1 | 사용자는 **특정 포켓몬 이름**으로 해당 포켓몬의 약점·저항·면역을 질의할 수 있어야 하며, 듀얼 타입의 복합 배율(4배/2배/0.5배/0.25배/0배)을 정확히 반영해야 한다. | 필수 |
| FR-05 | 도감 설명 검색 시 Vector(MMR) + BM25 + pg_trgm 세 채널을 RRF로 결합한 하이브리드 검색을 수행해야 한다. | 필수 |
| FR-06 | 사용자는 복합 조건(수치 + 묘사 + 진화 등)을 하나의 질문으로 질의할 수 있으며, 시스템은 이를 분해하여 여러 툴을 순서대로 호출해 통합 답변해야 한다. | 필수 |

### 3.2 Agent 동작

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-07 | Agent는 질문 유형에 따라 적절한 툴을 자율 선택해야 한다. | 필수 |
| FR-08 | Agent는 첫 번째 툴 호출 전에 사용자 질문을 핵심 검색 키워드 중심으로 재작성(Query Rewriting)해야 한다. | 필수 |
| FR-09 | Agent는 SQL 실행 오류 발생 시 오류 메시지를 분석하여 SQL을 수정·재시도해야 한다. | 필수 |
| FR-10 | Agent의 툴 호출은 최대 5회로 제한하며, 초과 시 수집된 정보로 최선의 답변을 생성해야 한다. | 필수 |
| FR-11 | Agent는 SELECT 이외의 SQL(INSERT, UPDATE, DELETE 등)을 차단해야 한다. | 필수 |
| FR-12 | Agent는 툴 실행 결과의 관련성을 LLM으로 평가(CRAG)하고, 관련성이 없으면 재검색을 유도하는 SystemMessage를 삽입해야 한다. | 필수 |
| FR-13 | 동일한 최솟값/최댓값을 가진 포켓몬이 여럿인 경우 전체를 반환해야 한다. (`ORDER BY ... LIMIT 1` 대신 `WHERE col = (SELECT MIN/MAX ...)` 서브쿼리 사용) | 필수 |

### 3.3 대화 관리

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-14 | 시스템은 대화 세션을 PostgreSQL에 영속 저장해야 한다. | 필수 |
| FR-15 | 사용자는 이전 세션을 재개하여 대화 문맥을 유지할 수 있어야 한다. | 필수 |
| FR-16 | 시스템은 각 메시지에서 사용된 툴 목록을 함께 저장해야 한다. | 선택 |
| FR-17 | 세션 목록은 최대 60개까지 조회할 수 있어야 한다. | 선택 |

### 3.4 데이터 초기화

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-18 | 신규 환경 구성 시 PGVector 임베딩 인덱스를 자동 생성하는 초기화 스크립트를 제공해야 한다. | 필수 |
| FR-19 | 이미 인덱싱된 데이터가 있을 경우 중복 생성 없이 건너뛰어야 한다 (idempotent). | 필수 |

## 4. 비기능 요구사항 (Non-Functional Requirements)

### 4.1 성능

| ID | 요구사항 | 기준값 |
|----|---------|--------|
| NFR-01 | 단순 SQL 조회 응답 시간 | ≤ 3초 |
| NFR-02 | 하이브리드 검색(Vector+BM25+pg_trgm) 포함 응답 시간 | ≤ 10초 |
| NFR-03 | 복합 툴(최대 5회) 호출 전체 응답 시간 | ≤ 30초 |
| NFR-04 | BM25 인덱스는 모듈 로드 시 1회만 빌드하고 이후 재사용해야 한다. | 필수 |

### 4.2 신뢰성 및 안전성

| ID | 요구사항 |
|----|---------|
| NFR-05 | SELECT 이외의 DML/DDL은 Tool 레벨에서 차단해야 한다. |
| NFR-06 | Neo4j 연결 실패 시 해당 툴만 오류를 반환하고 다른 툴은 정상 동작해야 한다. |
| NFR-07 | BM25 또는 pg_trgm 채널 실패 시 해당 채널을 제외하고 나머지 채널로 RRF를 수행해야 한다. |
| NFR-08 | 시스템은 툴 결과에 없는 정보를 LLM이 생성(환각)하지 않도록 System Prompt로 명시적으로 제약해야 한다. |

### 4.3 확장성

| ID | 요구사항 |
|----|---------|
| NFR-09 | 새로운 LLM 모델은 `MODELS` 딕셔너리에 항목 추가만으로 지원해야 한다. |
| NFR-10 | 새로운 Tool은 `@tool` 데코레이터 추가 및 `tools` 리스트 등록만으로 Agent에 통합되어야 한다. |

### 4.4 유지보수성

| ID | 요구사항 |
|----|---------|
| NFR-11 | 각 모듈(Agent, Neo4j, ChatHistory, Ingest)은 독립적으로 교체 가능해야 한다. |
| NFR-12 | 환경 변수는 `.env` 파일로 일원화하여 관리해야 한다. |

## 6. 제약사항 및 가정

- **LLM API 비용:** OpenAI API 사용에 따른 토큰 비용이 발생한다. CRAG 평가 및 Query Rewriting도 `gpt-4o-mini`를 호출하므로 호출당 추가 비용이 발생한다.
- **BM25 메모리:** `_bm25_index`는 프로세스 메모리에 상주하며, 서버 재시작 시 재빌드된다.
- **pg_trgm 확장:** `CREATE EXTENSION pg_trgm`이 PostgreSQL에 사전 활성화되어 있어야 한다.
- **웹 검색 미지원:** 외부 웹 검색(Tavily) 기능은 제거되었다. DB에 없는 정보는 "찾지 못했다"고 명시한다.
- **데이터 최신성:** DB에 적재된 포켓몬 데이터는 특정 시점 스냅샷이며 실시간 업데이트는 지원하지 않는다.
- **언어:** 한국어 질의·응답을 기본으로 하며, 영어 질의도 부분 지원한다.
- **동시 사용자:** 초기 버전은 단일 사용자 또는 소규모 동시 접속(≤ 10)을 가정한다.
