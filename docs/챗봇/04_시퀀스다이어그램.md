## 1. 전체 채팅 요청 흐름

```mermaid
sequenceDiagram
    actor User as 👤 사용자
    participant API as FastAPI Server
    participant CH as ChatHistory (PostgreSQL)
    participant AG as LangGraph Agent
    participant QR as Query Rewriter (GPT-4o-mini)
    participant TN as ToolNode
    participant CRAG as CRAG Check (GPT-4o-mini)
    participant PG as PostgreSQL (search_pokemon_db)
    participant HYB as Hybrid Search (Vector+BM25+pg_trgm+RRF)
    participant N4J as Neo4j (진화/타입/약점)

    User->>API: POST /chat { query, session_id?, model }
    API->>CH: load_messages(session_id)
    CH-->>API: 이전 대화 메시지 목록
    API->>AG: invoke({ messages, tool_call_count: 0 })

    Note over AG: SystemMessage + 대화 히스토리 + HumanMessage 조합

    AG->>QR: _rewrite_query(HumanMessage) — 첫 호출 시만
    QR-->>AG: 재작성된 질문 (핵심 키워드 명확화)

    AG->>AG: LLM 추론 → 툴 선택
    AG->>TN: tool_calls 전달 (tool_call_count < 5)
    TN->>PG: search_pokemon_db(sql) — 정형 데이터
    TN->>HYB: search_flavor_text(query) — 하이브리드 검색
    TN->>N4J: search_evolution_chain / search_type_relations / search_pokemon_weakness
    TN-->>AG: 툴 실행 결과

    AG->>CRAG: crag_check_node(state) — 결과 관련성 평가
    alt 관련성 낮음
        CRAG-->>AG: SystemMessage("재검색 유도")
        AG->>AG: LLM 재추론 → 다른 툴/검색어로 재시도
    else 관련성 충분
        CRAG-->>AG: 빈 메시지 (통과)
    end

    Note over AG: tool_call_count >= 5 이면 강제 답변 생성
    AG-->>API: 최종 답변

    API->>CH: save_message("user", query)
    API->>CH: save_message("assistant", answer, used_tools)
    API-->>User: { answer, used_tools, session_id }
```

## 2. Query Rewriting 상세 흐름

```mermaid
sequenceDiagram
    participant AG as LangGraph Agent
    participant QR as GPT-4o-mini (Rewriter)

    Note over AG: tool_call_count == 0 일 때만 실행
    AG->>AG: 메시지 목록에서 마지막 HumanMessage 탐색
    AG->>QR: "다음 포켓몬 질문의 핵심 의도를 유지하면서\n검색 키워드를 명확히 하세요...\n원본: {query}"
    QR-->>AG: 재작성된 질문 (한 문장)

    alt 재작성 성공 & 원본과 다름
        AG->>AG: HumanMessage 내용 교체
    else 실패 또는 동일
        AG->>AG: 원본 질문 그대로 유지 (fallback)
    end
```

## 3. Hybrid Search (search_flavor_text) 상세 흐름

```mermaid
sequenceDiagram
    participant TOOL as search_flavor_text
    participant VEC as PGVector (MMR)
    participant BM25 as BM25 (인메모리)
    participant TRGM as pg_trgm (PostgreSQL)
    participant DB as PostgreSQL (포켓몬명 조회)
    participant RRF as RRF 통합

    TOOL->>VEC: invoke(query) — k=20, fetch_k=50, λ=0.7
    VEC-->>TOOL: 벡터 유사 문서 20개 ("species_id: version 내용")

    TOOL->>DB: SELECT s.id, p.name WHERE s.id = ANY(valid_sids)
    DB-->>TOOL: {species_id → 포켓몬명} 매핑
    TOOL->>TOOL: 벡터 결과에 포켓몬명 부착 → vector_contents[]

    TOOL->>BM25: get_scores(query.split()) — 인메모리 인덱스
    BM25-->>TOOL: TOP 20 도감 설명 (score > 0)

    TOOL->>TRGM: SELECT ... WHERE similarity(content, query) > 0.05 LIMIT 20
    TRGM-->>TOOL: 트라이그램 유사 TOP 20

    TOOL->>RRF: _reciprocal_rank_fusion([vector, bm25, trgm], k=60)
    RRF-->>TOOL: 통합 정렬된 문서 목록
    TOOL-->>AG: 상위 5개 반환 (FLAVOR_TOP_N=5)
```

## 4. CRAG 흐름 (Corrective RAG)

```mermaid
sequenceDiagram
    participant TN as ToolNode
    participant CRAG as crag_check_node
    participant GRADER as GPT-4o-mini (Grader)
    participant AG as LangGraph Agent

    TN-->>CRAG: 툴 실행 완료 후 state 전달

    alt tool_call_count >= MAX-1 (4회 이상)
        CRAG-->>AG: 빈 메시지 (평가 생략, 루프 방지)
    else 빈 결과 / "찾을 수 없음" 포함
        CRAG-->>AG: SystemMessage("검색어를 바꾸거나 다른 툴로 재시도")
    else SQL 오류
        CRAG-->>AG: 빈 메시지 (SQL 오류는 Agent가 직접 처리)
    else 정상 결과
        CRAG->>GRADER: "검색 결과가 질문에 도움이 되면 YES, 무관하면 NO\n질문: ...\n결과: ..."
        GRADER-->>CRAG: "YES" or "NO"
        alt YES
            CRAG-->>AG: 빈 메시지 (통과)
        else NO
            CRAG-->>AG: SystemMessage("⚠️ 직전 검색 결과가 무관. 재검색하세요.")
        end
    end
    AG->>AG: LangGraph 루프 계속
```

## 5. SQL 오류 자동 복구 흐름

```mermaid
sequenceDiagram
    participant AG as LangGraph Agent
    participant TN as ToolNode
    participant PG as PostgreSQL

    AG->>TN: search_pokemon_db(잘못된 SQL)
    TN->>PG: 쿼리 실행 시도
    PG-->>TN: psycopg2.Error (컬럼명 오류 등)
    TN-->>AG: "SQL 오류: column 'atk' does not exist\n힌트: SQL을 수정 후 재시도하세요."

    Note over AG: tool_call_count += 1, CRAG 통과 (SQL 오류는 스킵)
    AG->>AG: 오류 메시지 분석 → 수정 SQL 생성
    AG->>TN: search_pokemon_db(수정된 SQL — 'attack' 사용)
    TN->>PG: 수정 쿼리 실행
    PG-->>TN: 정상 결과
    TN-->>AG: ✅ 결과 반환
```

## 6. 복합 질문 처리 흐름

```mermaid
sequenceDiagram
    actor User as 👤 사용자
    participant AG as LangGraph Agent
    participant TN as ToolNode
    participant PG as PostgreSQL
    participant HYB as Hybrid Search
    participant N4J as Neo4j

    User->>AG: "공격력 높고 귀여운 불꽃 타입과 그 진화 경로, 약점은?"

    Note over AG: Query Rewriting → "불꽃 타입 고공격 귀여운 포켓몬 진화 약점"

    AG->>TN: 1st call: [search_pokemon_db, search_flavor_text]
    TN->>PG: SELECT 불꽃타입 + 높은 attack (서브쿼리로 동점 처리)
    TN->>HYB: "귀여운 불꽃" Hybrid Search
    TN-->>AG: 두 결과 반환
    AG->>AG: CRAG 통과

    AG->>TN: 2nd call: [search_evolution_chain, search_pokemon_weakness]
    TN->>N4J: 후보 포켓몬 진화 체인 (LIMIT 1 + DISTINCT + 메가폼 필터)
    TN->>N4J: 후보 포켓몬 AGAINST 배율 조회
    TN-->>AG: 두 결과 반환
    AG->>AG: CRAG 통과

    AG-->>User: 통합 답변 (공격력 TOP + 귀여운 도감 + 진화 경로 + 약점)
```

## 7. 포켓몬 약점 조회 흐름

```mermaid
sequenceDiagram
    actor User as 👤 사용자
    participant AG as LangGraph Agent
    participant TN as ToolNode
    participant N4J as Neo4j

    User->>AG: "리자몽 약점은?"
    AG->>TN: search_pokemon_weakness("리자몽")
    TN->>N4J: MATCH (p:Pokemon)-[:HAS_TYPE]->(t)\nWHERE p.name CONTAINS "리자몽" → 타입 확인
    N4J-->>TN: types=["불꽃","비행"]
    TN->>N4J: MATCH (p)-[r:AGAINST]->(t)\nWHERE p.name CONTAINS "리자몽" → 전 배율 조회
    N4J-->>TN: [{바위,4.0},{물,2.0},{전기,2.0},{땅,0.0},...]
    TN->>TN: 버킷 분류 (very_weak/weak/resistant/immune)
    TN-->>AG: 🔍 [리자몽] 💀 4배: 바위 / ⚠️ 2배: 물,전기 / ✨ 면역: 땅
    AG-->>User: 리자몽은 바위에 4배 피해를 받습니다...
```

## 8. 임베딩 초기화 흐름 (ingest.py)

```mermaid
sequenceDiagram
    participant SCRIPT as ingest.py
    participant FS as 파일시스템 (flavor_text.json)
    participant VEC as PGVector (langchain_pg_embedding)
    participant OAI as OpenAI Embeddings API

    SCRIPT->>VEC: similarity_search("포켓몬", k=1) ← 기존 데이터 확인
    VEC-->>SCRIPT: 결과 있음 / 없음

    alt 데이터 없음 (최초 실행)
        SCRIPT->>FS: flavor_text.json 로드
        FS-->>SCRIPT: JSON 배열
        SCRIPT->>SCRIPT: Document 생성 ("species_id: version_name content")
        SCRIPT->>OAI: embed_documents()
        OAI-->>SCRIPT: vector(1536) 배열
        SCRIPT->>VEC: add_documents()
        VEC-->>SCRIPT: 인덱싱 완료
    else 데이터 있음
        SCRIPT->>SCRIPT: 건너뜀 (idempotent)
    end
```
