## 1. 전체 채팅 요청 흐름

```mermaid
sequenceDiagram
    actor User as 👤 사용자
    participant API as FastAPI Server
    participant CH as ChatHistory (PostgreSQL)
    participant AG as LangGraph Agent
    participant TN as ToolNode
    participant PG as PostgreSQL (search_pokemon_db)
    participant VEC as PGVector (search_flavor_text)
    participant N4J as Neo4j (진화/타입/약점)

    User->>API: POST /chat { query, session_id?, model }
    API->>CH: load_messages(session_id)
    CH-->>API: 이전 대화 메시지 목록

    API->>AG: invoke({ messages, tool_call_count: 0 })

    Note over AG: SystemMessage + 대화 히스토리 + HumanMessage 조합
    AG->>AG: LLM 추론 → 툴 선택

    alt 툴 호출 필요 (tool_call_count < 2)
        AG->>TN: tool_calls 전달
        TN->>PG: search_pokemon_db(sql)
        PG-->>TN: SQL 결과 JSON
        TN->>VEC: search_flavor_text(query)
        VEC-->>TN: 유사 도감 설명 목록
        TN-->>AG: 툴 실행 결과 반환
        AG->>AG: tool_call_count += 1
        AG->>AG: LLM 재추론 → 최종 답변 생성
    else 툴 불필요 or 횟수 초과
        AG->>AG: 직접 최종 답변 생성
    end

    AG-->>API: { messages, tool_call_count }
    API->>CH: save_message(session_id, "user", query)
    API->>CH: save_message(session_id, "assistant", answer, used_tools)
    CH-->>API: 저장 완료

    API-->>User: { answer, used_tools, session_id }
```

## 4. 복합 질문 처리 흐름 (SQL + 벡터 + 그래프 동시 활용)

```mermaid
sequenceDiagram
    actor User as 👤 사용자
    participant AG as LangGraph Agent (GPT-4o-mini)
    participant TN as ToolNode
    participant PG as PostgreSQL
    participant VEC as PGVector
    participant N4J as Neo4j

    User->>AG: "공격력 높고 귀여운 불꽃 타입 포켓몬과 그 진화 경로는?"

    Note over AG: 1차 LLM 추론 → 3개 툴 동시 호출 결정
    AG->>TN: [search_pokemon_db, search_flavor_text, search_evolution_chain] 동시 전달
    TN->>PG: SELECT 불꽃타입 + 높은 attack
    TN->>VEC: "귀여운 불꽃 느낌" MMR 검색
    TN->>N4J: EVOLVES_TO*1..3 (LIMIT 1 + DISTINCT) 진화 체인 조회

    PG-->>TN: 공격력 TOP 5 불꽃타입
    VEC-->>TN: 귀여운 도감 설명 TOP 20
    N4J-->>TN: 진화 경로 그래프 데이터

    TN-->>AG: 3개 툴 결과 통합

    Note over AG: 2차 LLM 추론 → 교차 분석 후 최종 답변
    AG-->>User: "리자드가 귀엽고 공격력도 높습니다...\n진화 경로: 파이리 → 리자드(Lv.16) → 리자몽(Lv.36)"
```

## 5. 포켓몬 약점 조회 흐름 (search_pokemon_weakness) ✨ NEW

듀얼 타입 복합 배율을 `AGAINST` 관계에서 직접 읽어 정확한 배율을 반환하는 흐름이다.

```mermaid
sequenceDiagram
    actor User as 👤 사용자
    participant AG as LangGraph Agent
    participant TN as ToolNode
    participant N4J as Neo4j

    User->>AG: "리자몽 약점이 뭐야?"

    Note over AG: 포켓몬 이름 명시 → search_pokemon_weakness 우선 선택
    AG->>TN: search_pokemon_weakness("리자몽")

    TN->>N4J: MATCH (p:Pokemon)-[:HAS_TYPE]->(t:Type)\nWHERE p.name CONTAINS "리자몽"\n→ 타입 목록 조회
    N4J-->>TN: pokemon="리자몽", types=["불꽃", "비행"]

    TN->>N4J: MATCH (p:Pokemon)-[r:AGAINST]->(t:Type)\nWHERE p.name CONTAINS "리자몽"\n→ 전체 공격 타입별 배율 조회
    N4J-->>TN: [{attack_type: "바위", multiplier: 4.0},\n{attack_type: "물", multiplier: 2.0}, ...]

    TN->>TN: 배율 기준으로 버킷 분류\n(4배 / 2배 / 0.5배 / 0.25배 / 0배)
    TN-->>AG: 🔍 [리자몽] 타입 상성\n💀 4배 약점: 바위\n⚠️ 2배 약점: 물, 전기\n✨ 면역: 땅

    AG-->>User: 리자몽은 바위 타입 공격에 4배 피해를 받습니다...
```

## 6. 임베딩 초기화 흐름 (ingest.py)

```mermaid
sequenceDiagram
    participant SCRIPT as ingest.py
    participant FS as 파일시스템 (flavor_text.json)
    participant VEC as PGVector (langchain_pg_embedding)
    participant OAI as OpenAI Embeddings API

    SCRIPT->>VEC: similarity_search("포켓몬", k=1) ← 기존 데이터 확인
    VEC-->>SCRIPT: 결과 있음 / 없음

    alt 데이터 없음 (최초 실행)
        SCRIPT->>FS: flavor_text.json 읽기
        FS-->>SCRIPT: JSON 배열 (도감 설명 목록)
        SCRIPT->>SCRIPT: Document 객체 생성 (species_id: version content)
        SCRIPT->>OAI: embed_documents(documents)
        OAI-->>SCRIPT: embedding 벡터 배열
        SCRIPT->>VEC: add_documents(documents + embeddings)
        VEC-->>SCRIPT: 인덱싱 완료
    else 데이터 있음
        SCRIPT->>SCRIPT: "이미 데이터 있음" 로그 출력 후 종료
    end
```
