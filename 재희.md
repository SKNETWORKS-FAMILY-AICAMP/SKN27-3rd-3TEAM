# Streamlit Cloud 배포 시 Neo4j 연결 문제 해결 기록

> 팀빌더 AI박사 기능이 배포 환경에서 동작하지 않던 문제를 해결한 기록입니다.

---

## 증상

- 배포 URL: https://skn27-3rd-3team.streamlit.app
- 팀빌더 AI박사 버튼 클릭 시 아무 응답 없음 / 오류 발생
- `/api/v1/team-builder/pokemon-options` → HTTP 500 에러
- `/api/v1/pokemon/` (PostgreSQL) → 정상 동작

---

## 원인 분석

### 1. Streamlit Cloud는 Neo4j를 직접 실행할 수 없다

Streamlit Cloud는 Python/Streamlit 앱만 실행하는 환경입니다.
Neo4j 데이터베이스 서버를 Streamlit Cloud에 올리는 것은 불가능합니다.

### 2. 아키텍처 구조

```
Streamlit Cloud (frontend)
    └─ BACKEND_URL → Railway 백엔드 호출

Railway (backend - FastAPI)
    ├─ PostgreSQL → 정상 연결
    └─ Neo4j → 환경변수 미설정으로 500 에러 발생

Railway (Neo4j DB)
    └─ 데이터는 이미 적재되어 있었음
```

### 3. 핵심 원인: Railway 백엔드에 Neo4j 환경변수가 없었다

Railway 백엔드 서비스에 Neo4j 접속 정보가 설정되어 있지 않아서,
백엔드가 Neo4j에 연결을 시도할 때마다 실패하고 있었습니다.

### 4. 변수명 함정

| 출처 | 변수명 |
|---|---|
| Neo4j AuraDB 제공 | `NEO4J_USERNAME` |
| 코드에서 읽는 변수 (`neo4j_client.py`) | `NEO4J_USER` 또는 `GRAPH_DB_USER` |

AuraDB에서 제공하는 `NEO4J_USERNAME`을 그대로 쓰면 코드가 읽지 못합니다.
반드시 `NEO4J_USER` 또는 `GRAPH_DB_USER`로 설정해야 합니다.

---

## 해결 방법

### Railway 백엔드 Variables에 아래 환경변수 추가

```
GRAPH_DB_URI=bolt://yamabiko.proxy.rlwy.net:53387
GRAPH_DB_USER=neo4j
GRAPH_DB_PASSWORD=test1234
```

저장 후 Railway가 자동 재시작되면 팀빌더 AI박사 정상 동작.

---

## 환경변수 전체 정리

### Streamlit Cloud Secrets

```toml
BACKEND_URL = "https://backend-production-8f80.up.railway.app"
```

### Railway 백엔드 Variables

```
# PostgreSQL
DATABASE_URL=postgresql://...

# Neo4j (Railway 내부 Neo4j)
GRAPH_DB_URI=bolt://yamabiko.proxy.rlwy.net:53387
GRAPH_DB_USER=neo4j
GRAPH_DB_PASSWORD=test1234

# OpenAI 등 API 키
OPENAI_API_KEY=...
```

### 로컬 .env

```env
BACKEND_URL = "https://backend-production-8f80.up.railway.app"
GRAPH_DB_URI = "bolt://yamabiko.proxy.rlwy.net:53387"
GRAPH_DB_USER = "neo4j"
GRAPH_DB_PASSWORD = "test1234"
```
